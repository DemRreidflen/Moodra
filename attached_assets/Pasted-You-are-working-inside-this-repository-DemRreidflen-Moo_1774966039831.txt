- The actual printable HTML is generated in server/routes.ts inside the route:
  app.get("/api/books/:id/export/pdf-html", ...)
- package.json already includes:
  hypher
  hyphenation.ru
  hyphenation.uk
  hyphenation.de
  hyphenation.en-us
  pagedjs
- Right now Russian/Ukrainian words may wrap at line/page ends without a visible hyphen.
- English and German must remain unchanged unless absolutely necessary.

Goal:
Implement robust server-side hyphenation preprocessing for RU and UK only, using soft hyphens (\u00AD), so that long words break correctly in the generated printable HTML and show a visible hyphen at line breaks in PDF/print output.

What to change:

1. In server/routes.ts, add imports for:
   - Hypher
   - hyphenation.ru
   - hyphenation.uk

2. Create reusable helpers near the PDF HTML export code:
   - detect whether current export language is ru or uk
   - hyphenate plain text using Hypher and the corresponding patterns
   - preserve HTML tags while hyphenating only text nodes
   - do NOT hyphenate:
     - headings h1/h2/h3
     - TOC entries
     - chapter titles
     - URLs
     - emails
     - file paths
     - code-like strings
     - very short words
   - avoid double-inserting soft hyphens if text was already processed

3. The best implementation strategy:
   - add a function like hyphenateTextContentForPdf(text: string, lang: "ru" | "uk"): string
   - add a function like hyphenateHtmlPreservingTags(html: string, lang: "ru" | "uk"): string

   - for semantic paragraph-like blocks (paragraph/default, quote, idea, argument, counterargument, hypothesis, question, observation, etc.) it is okay to hyphenate the inner text
   - for h1/h2/h3/heading and toc/chapter labels, skip hyphenation
