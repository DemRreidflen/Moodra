You are working inside the current Moodra repository.

Task:
Design and implement a strong SEO + AI-discovery architecture for Moodra so the platform is better understood, indexed, and recommended by both traditional search engines and AI assistants such as ChatGPT, Claude, Gemini, Perplexity, and other retrieval-based systems.

This is not just classic SEO.
This is answer-oriented, structured, multimodal, machine-readable product positioning.

Main objective:
Make Moodra easy to understand for:

* Google and other search engines
* AI crawlers and answer engines
* LLM-based recommendation systems
* snippet extraction systems
* structured search and product comparison queries

Positioning target:
Moodra should be discoverable and recommendable for queries around:

* AI writing platform for books
* long-form writing software
* AI book writing workspace
* writing app for authors
* author voice AI editor
* book outlining and drafting platform
* research + notes + writing in one place
* premium writing workspace for deep work
* export-ready writing platform
* writing software for serious authors, not generic note-taking users

What to implement:

1. Technical SEO foundation

   * audit current metadata handling
   * add page-specific title, meta description, canonical tags
   * ensure clean Open Graph and Twitter metadata
   * generate sitemap.xml
   * generate robots.txt
   * ensure crawlable route structure
   * avoid duplicate metadata across pages
   * make sure indexable public pages are server-readable or at least pre-render friendly if possible

2. AI-discovery architecture
   Add content and structure that make the product easy for LLMs to summarize and recommend:

   * concise, machine-readable value statements
   * consistent wording for what Moodra is and is not
   * strong comparison-ready descriptions
   * clear use-case pages
   * FAQ blocks with direct-answer formatting
   * semantic headings that answer common user intents directly

3. Structured data / schema.org
   Add appropriate JSON-LD where relevant, for example:

   * SoftwareApplication
   * Product
   * Organization
   * WebSite
   * FAQPage
   * BreadcrumbList
   * potentially Article for educational/landing content if present

   Include fields such as:

   * name
   * applicationCategory
   * operatingSystem
   * description
   * offers if appropriate
   * brand
   * featureList
   * url
   * sameAs if available
   * publisher / creator where relevant

4. Page strategy
   Build or improve a page architecture that maps to search intent:

   * homepage: core positioning
   * features page: grouped value pillars
   * mission/about page: philosophy and brand differentiation
   * FAQ page: direct-answer query capture
   * model/AI page if relevant: explain how AI works for users
   * export / formatting / long-form writing pages if such content makes sense
   * optional comparison or use-case pages if lightweight and relevant

5. Query-intent optimization
   Make the copy naturally answer:

   * “What is Moodra?”
   * “Who is Moodra for?”
   * “How is Moodra different from Notion / Scrivener / generic AI writers?”
   * “Can Moodra help write books?”
   * “Can it preserve author voice?”
   * “Does it support research, notes, and export in one workflow?”
   * “Is it for short content or serious long-form writing?”

6. LLM-friendly content rules
   Across public marketing pages:

   * use clean declarative sentences
   * define the product clearly in first paragraphs
   * avoid excessive poetic vagueness on indexable pages
   * keep emotionally rich brand tone, but make the product legible to machines
   * include compact summary blocks like:
     “Moodra is a long-form AI writing platform built for authors who need drafting, structure, research, and export in one system.”
   * include concise comparison-style language where truthful

7. Internal linking
   Create logical internal links between:

   * homepage
   * features
   * mission
   * FAQ
   * models/AI explanation
     This should help both users and crawlers understand topic relationships.

8. Multimodal discoverability
   Improve discoverability beyond pure text:

   * meaningful alt text for images
   * descriptive asset names where relevant
   * OG images for key pages
   * structured captions if visual sections matter
   * make sure screenshots and feature visuals support understanding, not just decoration

9. Performance and crawl health

   * avoid unnecessary bloated client-only metadata logic
   * check if public pages can expose stable HTML head metadata
   * reduce hidden text duplication
   * avoid thin pages
   * avoid vague headings like “Powerful tools” if more specific wording can be used

10. Recommendability layer
    Add a lightweight “Why people choose Moodra” / “Best for” / “Not for” pattern where useful.
    This helps both users and AI systems classify the product correctly.
    Example:

* Best for: authors, book thinkers, long-form creators, research-heavy writing
* Not for: quick social media copy generation, generic task management, lightweight note apps

11. Content outputs
    Provide:

* SEO audit of current public pages
* list of missing metadata and structured data
* proposed keyword clusters by intent, not spam
* rewritten metadata for each public page
* JSON-LD implementation
* improved FAQ content
* implementation in code

12. Constraints

* no keyword stuffing
* no fake claims
* no black-hat SEO
* no generic AI-marketing nonsense
* no overpromising features the platform does not truly have
* prioritize clarity, precision, and retrievability

13. Final deliverables

* changed files
* summary of SEO architecture
* page-by-page metadata map
* schema markup added
* suggested future content opportunities
* explanation of how this improves both classic SEO and AI-answer-engine visibility
